Executive Summary

The ResistanceZero portfolio website demonstrates excellent SEO fundamentals. The site has comprehensive meta tags, well-structured XML sitemap, extensive structured data (JSON-LD), RSS feed, canonical URLs, proper robots.txt, and consistent Open Graph/Twitter Card metadata across all 30+ pages. The technical SEO foundation is among the strongest areas of the site.

The main improvement opportunities are in performance optimization (inline CSS bloat, render-blocking resources), missing hreflang for bilingual content (Indonesian/English), and incomplete image SEO on some infrastructure pages. The site is well-positioned for search visibility in the data center engineering niche.

Nginx gzip is enabled for all text-based content types with a minimum size of 256 bytes. This reduces transfer size by 60–80% for CSS, JS, HTML, and SVG files.
nginx.conf:30–32
Pass Font Loading Prevents Layout Shift
Fonts use display=swap and preconnect to fonts.googleapis.com and fonts.gstatic.com. The preload-as-style pattern with onload fallback ensures non-render-blocking font loading.
index.html:63–66
High Massive Inline CSS Blocks Increase HTML Payload
Article pages contain thousands of lines of inline CSS for Pro Mode calculators, charts, and interactive components. This CSS is included in every page load even for free users who do not see Pro features. Combined with the 5000+ line styles.css, the total CSS payload per article page is estimated at 100–200KB (uncompressed). This impacts First Contentful Paint (FCP) and Time to Interactive (TTI).
Recommendation: Move Pro Mode CSS to an external pro-mode.css file loaded only when needed. Use critical CSS extraction for above-the-fold content inline, and defer the rest. The styles.css file should be split into base.css (critical) and components.css (deferred).
Medium Google Analytics Blocks Initial Rendering
The Google Analytics script is the first element in <head>, loaded before meta tags. While it uses async, the inline gtag initialization script executes synchronously. Moving it to the end of <body> or using defer would improve First Contentful Paint.
index.html:5–12
Recommendation: Move the Google Analytics script block to just before </body>, or use the Google Tag Manager with delayed initialization (load after DOMContentLoaded).
Medium Universal CSS Transition Causes CLS Risk
The rule *, *::before, *::after { transition: background-color 0.3s, border-color 0.3s, box-shadow 0.3s; } causes all elements to animate on theme switch. During initial paint, if the theme is loaded from localStorage, elements may visually transition from default to dark theme, causing a perceptible Cumulative Layout Shift (CLS) and visual jank.
styles.css:147–149
Recommendation: Apply theme before first render (inline script in <head> that sets data-theme before CSS loads). Remove the universal transition rule and apply transitions only to theme-sensitive elements.
Low Lazy Loading on Above-Fold Avatar
The navbar avatar uses loading="lazy" despite being visible on initial load. This delays rendering of a visible LCP-adjacent element. The 9 total lazy-loaded images on the homepage are otherwise appropriately applied to below-fold content.
index.html:156
Recommendation: Remove loading="lazy" from the navbar avatar. Keep lazy loading on below-fold images.

Medium Horizontal Scroll Risk on Complex Article Layouts
The bento grid career timeline (5 cards in a row), calculator tables, and Pro Mode panels may cause horizontal overflow on narrow mobile screens if responsive breakpoints are not exhaustively tested. Google's Mobile-Friendly Test penalizes pages with horizontal scrolling.
Recommendation: Run Google's Mobile-Friendly Test on each article page. Add overflow-x: hidden to the body as a safety net, and ensure all grid/table layouts have mobile breakpoints.
Medium Heavy JavaScript Impacts Mobile TTI
Article pages with Pro Mode include heavy JavaScript for Monte Carlo simulations (10K iterations), canvas chart rendering, SVG generation for PDF export, and real-time sensitivity analysis. On mobile devices, this JavaScript execution can push Time to Interactive (TTI) well beyond the 3.8s "good" threshold. Google uses mobile-first indexing.
Recommendation: Defer Pro Mode JavaScript loading until the user explicitly activates Pro features. Use <script defer> or dynamic import() for calculator modules. Consider a Service Worker for offline caching of heavy scripts.
