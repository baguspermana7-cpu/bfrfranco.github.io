CAPEX 1. Architecture Overview

The CAPEX Calculator estimates total data center construction capital expenditure using a parametric cost model. It operates in three modes:

Core Formula:
Cost[category] = baseFactor[$/kW] × IT_Load[kW] × multiplier_chain × advGlobalMult
Total = Σ(all categories) + softCosts + contingency + FOM + renewables

CAPEX 7. Core Calculation Algorithm

// Step 1: Resolve all multipliers
advGlobalMult = yearMult * marketMult * deliveryMult * contractorMult  // (1.0 in Simple mode)

// Step 2: Calculate each cost category
for each category in costFactors:
    multiplier = locationMult
    // Apply category-specific multipliers:
    if building:  multiplier *= buildMult * rackMult * floorM * siteM
    if electrical: multiplier *= redMult * rackMult * powerDistM * transformerM
    if ups:       multiplier *= redMult * rackMult * upsMult
    if generator: multiplier *= redMult * fuelMult * genMult
    if cooling:   multiplier *= coolMult * rackMult
    if testing:   multiplier *= regionTest * redundancyTest
    if permits:   multiplier *= regionPermits

    costs[category] = baseFactor * IT_Load * multiplier * advGlobalMult

// Step 3: Per-rack fixed costs (Advanced only)
racks = ceil(IT_Load / rackKw[rackType])
costs.electrical += racks * pduPerRack * advGlobalMult * locMult
costs.network += racks * cablingPerRack * advGlobalMult * locMult + fiberCost

// Step 4: Soft costs (Advanced only)
softCosts.design = subtotal * (designPct / 100)
softCosts.pm = subtotal * (pmPct / 100)

// Step 5: Green certification premium
greenPremium = subtotal * (greenMult - 1.0)

// Step 6: Contingency
contingency = subtotal * (contingencyPct / 100)  // Default 5%

// Step 7: Add FOM + Renewables
TOTAL = subtotal + softCosts + greenPremium + contingency + fomTotal + renewableCost

18.3 OEM Factor by Staffing Model

In-House: oemFactor = 1.0 (self-maintained)
Full Contractor: oemFactor = system.oemMultiplier (2.0-2.5x)
Partial: oemFactor = 1.0 + (oemMult - 1.0) × contractorPct
Key Insight: OEM contractors charge 2-3x more due to: travel/mobilization, profit margin, specialized parts inventory, SLA guarantees, and overhead. This is the single largest variable in maintenance OPEX and the primary driver for the in-house vs. contractor cost differential.

19.3 Labor Metrics

19.4 Carbon Metrics

co2Tonnes = (annualKWh × gridEmissionFactor[country]) / 1000
carbonCost = co2Tonnes × carbonPrice[region]
CUE = (co2Tonnes × 1000) / (IT_Load × 8760 × utilization)

19.5 Fixed vs Variable Analysis

fixedCost = staffing + insurance + software
variableCost = energy + utilities + other
operatingLeverage = variableCost / totalOPEX

22.2 Model Limitations

  • Parametric estimation — actual costs vary with site conditions, procurement strategy, and market timing
  • City data is sourced from industry reports; actual quotes may differ by 15-25%
  • Equipment brand database is indicative — vendor selection requires RFP process
  • Staffing model uses generic FTE ratios — actual needs depend on scope, SLA, and organizational structure
  • Carbon pricing uses regional averages; actual schemes (ETS, carbon tax) vary significantly
  • TCO projections assume steady-state escalation; actual costs may be step-function (contract renewals, equipment replacement)

22.3 Recommended Validation Approach

  1. Cross-reference total $/kW against T&T DCCI published ranges for the target city
  2. Validate cooling energy with PUE measurement data from similar facilities
  3. Compare staffing FTE against Uptime Institute benchmarks (staff/MW)
  4. Verify maintenance costs against actual OEM quotes for major systems
  5. Run scenario A/B to bound uncertainty (optimistic vs conservative)
