# Pokemon Assets Folder

## Folder Structure
```
assets/Pokemon/
├── README.md           ← this file
├── pokemon-db.json     ← full Pokemon database (id, name, slug, types)
└── sprites/            ← PNG files (download manually or via script)
    ├── pikachu.png
    ├── bulbasaur.png
    └── ...
```

## Sprite URL Format
Online sprites (no download needed — game uses these directly):
- **pokemondb HD**: `https://img.pokemondb.net/sprites/home/normal/{slug}.png`
- **PokeAPI backup**: `https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/{id}.png`

## Download All Sprites (optional, for offline use)
```bash
# Run from this folder
while IFS=',' read id slug; do
  wget -q "https://img.pokemondb.net/sprites/home/normal/${slug}.png" -O "sprites/${slug}.png"
done < sprite-list.csv
```

## Usage in Game
All Pokemon data is embedded in `index.html` as `POKEMON_DB` array.
Sprite URLs are generated by `pokeSprite(slug)` and `pokeSpriteBackup(id)` functions.

## Adding New Pokemon
Add to `POKEMON_DB` in index.html:
```js
{id: 1234, name: 'NewPokemon', slug: 'newpokemon', type: 'fire', type2: 'dragon'}
```
